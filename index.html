Skills do Claude
Comandos avançados baseados no artigo de Skills. Use no Claude Code ou como referência de prompts.
O que são Skills
Skills são arquivos SKILL.md que ensinam o Claude a fazer tarefas específicas com processo definido. Em vez de reexplicar seu workflow toda sessão, você instala o processo uma vez e reutiliza para sempre. Para você, o mais útil é usar os princípios dessas skills como comandos nos seus prompts.

TDD — Test Driven Development
Escreve testes antes do código. Garante que cada feature funciona antes de avançar.
Comando:
"Antes de codar, escreva os testes que provam que essa feature funciona. Só depois implemente o código mínimo para passar nos testes."

Systematic Debugging
4 fases: reproduzir → causa raiz → única correção → verificar. Proíbe tentativa e erro.
"Siga debug sistemático: 1) reproduza com caso mínimo, 2) encontre causa raiz, 3) aplique UMA correção, 4) verifique com testes. Não tente múltiplas coisas."
🏗️
Grill Me — Perguntas antes de codar
Força o Claude a fazer perguntas difíceis sobre edge cases, decisões de dados e falhas antes de construir.
"Antes de qualquer código, me faça perguntas difíceis sobre essa feature — uma por vez — até resolver toda a árvore de decisão."
